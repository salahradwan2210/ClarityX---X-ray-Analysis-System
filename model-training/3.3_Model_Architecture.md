# 3.3 Model Architecture

## 3.3.1 Overview

Our chest X-ray analysis system employs an advanced deep learning architecture designed specifically for the challenges of medical image analysis. The architecture combines a powerful backbone network with specialized modules for classification, localization, and metadata integration. This design enables simultaneous disease classification and anatomical localization while maintaining high clinical accuracy.

## 3.3.2 Backbone Network

### 3.3.2.1 ConvNeXt Large

The foundation of our model is the ConvNeXt Large architecture, which represents a modern evolution of convolutional neural networks with several advantages for medical imaging:

- **Hierarchical Feature Extraction**: ConvNeXt's design progressively captures features at multiple scales, from fine-grained texture details to larger anatomical structures.

- **Large Receptive Field**: The network's depth and architecture create an extensive receptive field capable of capturing long-range dependencies within chest radiographs.

- **Parameter Efficiency**: Despite its large capacity (approximately 198M parameters), ConvNeXt achieves higher performance per parameter compared to earlier architectures.

- **Modern Building Blocks**: The backbone incorporates advanced components including:
  - Depthwise separable convolutions
  - Layer normalization
  - GELU activation functions
  - Global response normalization

The final feature map from the backbone has dimensions of 1536 channels, providing a rich representation of the input chest X-ray.

## 3.3.3 Attention Mechanism

We enhance the backbone's feature extraction capabilities with a specialized channel attention mechanism:

```
self.attention = nn.Sequential(
    nn.LayerNorm(in_features),
    nn.Linear(in_features, in_features // 8),
    nn.GELU(),
    nn.Dropout(0.05),
    nn.Linear(in_features // 8, in_features),
    nn.Sigmoid()
)
```

This attention module:

1. Normalizes the input feature map
2. Compresses the channel dimension by a factor of 8
3. Applies non-linear transformation with GELU activation
4. Expands back to the original dimension
5. Produces channel-wise attention weights via sigmoid activation

The resulting attention weights modulate the feature map, emphasizing informative channels and suppressing less relevant ones. This mechanism significantly improves the model's ability to focus on pathology-relevant features.

## 3.3.4 Multi-Task Learning Heads

Our architecture implements a multi-task learning approach with specialized heads for different but related tasks:

### 3.3.4.1 Classification Head

The classification head predicts the probability of 14 distinct thoracic pathologies:

```
self.combined_fc = nn.Sequential(
    nn.LayerNorm(in_features + 128),
    nn.Dropout(0.1),
    nn.Linear(in_features + 128, in_features // 2),
    nn.GELU(),
    nn.LayerNorm(in_features // 2),
    nn.Dropout(0.05),
    nn.Linear(in_features // 2, num_classes)
)
```

Key features of the classification head include:

- **Two-Stage Design**: The multi-layer architecture enables more complex decision boundaries.
- **Regularization**: Strategic dropout layers (rates of 0.1 and 0.05) prevent overfitting.
- **Normalization**: Layer normalization between stages improves training stability.
- **Multi-label Output**: The final layer produces logits for all 14 pathologies without softmax, allowing for multiple concurrent diagnoses.

### 3.3.4.2 Localization Head

For 8 of the 14 pathologies, our model provides spatial localization through bounding box predictions:

```
self.localization_head = nn.Sequential(
    nn.LayerNorm(in_features),
    nn.Dropout(0.05),
    nn.Linear(in_features, in_features // 2),
    nn.GELU(),
    nn.LayerNorm(in_features // 2),
    nn.Dropout(0.05),
    nn.Linear(in_features // 2, in_features // 4),
    nn.GELU(),
    nn.LayerNorm(in_features // 4),
    nn.Linear(in_features // 4, bbox_classes * 4)
)
```

The localization head:

- Processes attended feature maps from the backbone
- Progressively refines the representation through multiple layers
- Outputs 32 values representing 4 coordinates (x, y, width, height) for each of the 8 pathologies
- Applies sigmoid activation to normalize coordinates to [0,1] range

## 3.3.5 Metadata Integration

A unique aspect of our architecture is the integration of patient metadata with imaging features:

```
self.metadata_branch = nn.Sequential(
    nn.Linear(metadata_features, 64),
    nn.LayerNorm(64),
    nn.ReLU(),
    nn.Linear(64, 128),
    nn.LayerNorm(128),
    nn.ReLU(),
    nn.Linear(128, 128),
    nn.LayerNorm(128),
    nn.ReLU()
)
```

This branch:

1. Processes 3 metadata features (age, gender, view position)
2. Progressively expands the representation to 128 dimensions
3. Applies ReLU activations and layer normalization for stable training
4. Concatenates the resulting metadata features with image features before classification

The metadata integration improves model performance by incorporating clinically relevant non-imaging information that provides context for accurate diagnosis.

## 3.3.6 Forward Pass Workflow

The model's forward pass integrates all components in a coordinated workflow:

1. **Feature Extraction**: The input image passes through the ConvNeXt backbone to produce a 1536-dimensional feature vector.

2. **Attention Application**: The channel attention module weighs feature channels by importance, producing attended features.

3. **Metadata Processing**: Patient metadata is processed through the metadata branch in parallel.

4. **Localization Prediction**: The localization head processes attended features to predict bounding boxes.

5. **Feature Fusion**: Attended image features are concatenated with processed metadata features.

6. **Classification**: The combined feature vector passes through the classification head to predict pathology probabilities.

The forward pass produces two primary outputs:
- Disease classification scores (14 values)
- Bounding box coordinates (8 × 4 = 32 values)

## 3.3.7 Loss Functions

To enable effective multi-task learning, we employ specialized loss functions:

### 3.3.7.1 Classification Loss

We implement a modified Focal Loss with label smoothing:

```
class FocalLoss(nn.Module):
    def __init__(self, alpha=0.6, gamma=2.0, label_smoothing=0.1, reduction='mean'):
        super(FocalLoss, self).__init__()
        self.alpha = alpha
        self.gamma = gamma
        self.label_smoothing = label_smoothing
        self.reduction = reduction
        
    def forward(self, inputs, targets):
        if self.label_smoothing > 0:
            targets = targets * (1 - self.label_smoothing) + self.label_smoothing / inputs.shape[1]
        BCE_loss = F.binary_cross_entropy_with_logits(inputs, targets, reduction='none')
        pt = torch.exp(-BCE_loss)
        F_loss = self.alpha * (1 - pt) ** self.gamma * BCE_loss
        if self.reduction == 'mean':
            return F_loss.mean()
        else:
            return F_loss
```

This loss function:
- Addresses class imbalance through the alpha parameter (0.6)
- Focuses on hard examples with the gamma parameter (2.0)
- Prevents overconfident predictions with label smoothing (0.1)
- Works effectively for multi-label classification

### 3.3.7.2 Localization Loss

For bounding box prediction, we combine two complementary loss functions:

1. **Smooth L1 Loss**: For coordinate regression
2. **IoU Loss**: For optimizing overlap between predicted and ground truth boxes

The weighted combination (0.4 × L1 + 0.6 × IoU) provides better localization performance than either loss alone.

### 3.3.7.3 Combined Loss

The final training objective combines classification and localization losses with a configurable weighting:

```
total_loss = cls_loss + BBOX_LOSS_WEIGHT * bbox_loss
```

The BBOX_LOSS_WEIGHT parameter (set to 1.0 in our fine-tuned model) balances the importance of accurate classification versus precise localization.

## 3.3.8 Model Complexity and Performance

The complete model architecture includes:

- Parameters: ~200M (majority in the ConvNeXt backbone)
- FLOPs: ~25G for a single forward pass at 512×512 resolution
- Memory: ~2.5GB GPU memory for inference with batch size 1

Despite this complexity, inference time remains clinically viable at ~0.3 seconds per image on modern GPUs.

## 3.3.9 Implementation Details

The model is implemented in PyTorch with the following key components:

- **Backbone**: Leverages the TIMM library implementation of ConvNeXt Large
- **Gradient Checkpointing**: Enabled to reduce memory requirements during training
- **Mixed Precision**: Uses FP16 computation with AMP for faster training
- **Gradient Accumulation**: Accumulates gradients across multiple steps to simulate larger batch sizes
- **Warm Restarts**: Implements learning rate cycling for improved convergence

## 3.3.10 Architecture Evolution

The model architecture evolved through extensive experimentation:

1. **Baseline**: Standard ConvNeXt with classification head (AUROC: 0.923)
2. **Added Attention**: Incorporating channel attention (AUROC: 0.945)
3. **Metadata Integration**: Adding patient metadata (AUROC: 0.952)
4. **Multi-task Learning**: Adding localization (AUROC: 0.965)
5. **Fine-tuned Architecture**: Optimized hyperparameters and enhanced localization (AUROC: 0.969)

Each architectural enhancement was validated through rigorous experimentation, with improvements measured by mean AUROC on the validation set.

In summary, our model architecture represents a comprehensive approach to chest X-ray analysis, combining state-of-the-art deep learning techniques with domain-specific optimizations for medical imaging. The resulting system achieves high performance in both classification and localization tasks, with a mean AUROC of 0.969 across 14 thoracic pathologies.
