# 3.2 Data Preprocessing

## 3.2.1 Overview

Data preprocessing is a critical step in developing an effective chest X-ray classification and localization system. The NIH Chest X-ray dataset presents several challenges that must be addressed through careful preprocessing techniques. These include image quality variations, diverse acquisition protocols, metadata inconsistencies, and inherent class imbalance. Our preprocessing pipeline addresses these challenges systematically to ensure optimal model performance.

## 3.2.2 Image Preprocessing

### 3.2.2.1 Resolution Standardization

All images are resized to a uniform resolution of 512×512 pixels. This standardization serves multiple purposes:

- Ensures consistent input dimensions for the neural network
- Preserves sufficient anatomical detail for accurate diagnosis
- Balances computational efficiency with diagnostic accuracy
- Allows for effective batch processing during training

The resizing operation uses area interpolation (cv2.INTER_AREA) to minimize artifacts when downsampling from higher resolution images.

### 3.2.2.2 Intensity Normalization

Radiographic images exhibit significant variations in intensity distributions due to differences in:

- X-ray exposure settings
- Detector sensitivity
- Patient characteristics (body mass, tissue density)
- Hospital-specific acquisition protocols

To address these variations, we implement a comprehensive normalization strategy:

1. Pixel values are normalized using the ImageNet mean and standard deviation: 
   - Mean: [0.485, 0.456, 0.406]
   - Standard deviation: [0.229, 0.224, 0.225]

2. This normalization maps pixel values to a standardized range that aligns with the pretrained ConvNext backbone, which was initially trained on ImageNet.

### 3.2.2.3 Data Augmentation

To improve model generalization and robustness, we implement an extensive data augmentation pipeline using the Albumentations library. Our augmentation strategy includes:

**Geometric Transformations:**
- Horizontal flipping (p=0.5): Simulates alternative patient positioning
- Random rotation (±10° with p=0.2): Accounts for patient positioning variations
- Grid distortion (p=0.1): Simulates subtle projection variations
- Optical distortion (p=0.1): Mimics acquisition artifacts

**Intensity Transformations:**
- Brightness and contrast adjustments (±15% with p=0.4): Simulates exposure variations
- Color jittering (p=0.3): Enhances robustness to scanner variations
- Gaussian noise (var_limit=5.0-20.0, p=0.1): Simulates detector noise

**Regional Transformations:**
- Coarse dropout (p=0.2): Simulates occlusions or artifacts

This augmentation pipeline is applied exclusively during training to enhance model robustness. Validation and inference procedures use only standardized resizing and normalization without augmentations.

## 3.2.3 Metadata Preprocessing

The NIH dataset provides several metadata fields that can enhance diagnostic performance, including:

- Patient age
- Patient gender
- View position (e.g., PA, AP, lateral)

These fields require preprocessing to handle missing values and normalize distributions:

### 3.2.3.1 Age Normalization

1. Patient age is extracted as a numeric value and clipped to a maximum of 110 years to remove outliers.
2. Missing age values are imputed using the mean age across the dataset.
3. Age values are normalized to the [0,1] range by dividing by 100, creating a scaled representation suitable for neural network input.

### 3.2.3.2 Gender Encoding

1. Gender information is standardized with consistent capitalization and whitespace removal.
2. Missing gender values are replaced with an "Unknown" category.
3. A label encoder transforms categorical gender values into numeric indices.
4. The resulting indices are one-hot encoded for network compatibility.

### 3.2.3.3 View Position Encoding

1. View position strings are standardized with consistent formatting.
2. Missing values are replaced with an "Unknown" category.
3. A label encoder transforms the categorical positions into numeric indices.
4. The resulting indices are one-hot encoded for network input.

## 3.2.4 Label Processing

### 3.2.4.1 Multi-label Classification

The NIH dataset presents a multi-label classification problem where each image may contain multiple pathologies. Our label processing involves:

1. Parsing the "Finding Labels" field, which contains pipe-separated pathology names.
2. Converting these labels into a binary vector where each element represents the presence (1) or absence (0) of a specific pathology.
3. The "No Finding" class is handled specially - it's represented by an all-zeros vector for the 14 pathology classes.

### 3.2.4.2 Bounding Box Preprocessing

For the 8 pathologies with bounding box annotations:

1. Bounding box coordinates are extracted from the BBox_List_2017.csv file.
2. Coordinates are normalized to [0,1] range by dividing by the original image dimensions (1024×1024).
3. Invalid or missing bounding box annotations are represented as zeros.
4. A mask is created to indicate the presence of valid bounding box annotations for each pathology.

## 3.2.5 Handling Class Imbalance

The NIH dataset exhibits significant class imbalance, with common conditions like "Infiltration" appearing frequently while others like "Hernia" are rare. Our preprocessing pipeline addresses this imbalance through:

### 3.2.5.1 Weighted Sampling

1. Calculating class weights inversely proportional to class frequency: `weight(c) = 1 / count(c)`
2. Computing per-sample weights based on the maximum class weight of present pathologies.
3. Implementing a `WeightedRandomSampler` that oversamples minority classes and undersamples majority classes.
4. Special handling for "No Finding" cases with a moderate weight multiplier to prevent overfitting to either normal or pathological cases.

### 3.2.5.2 Controlled Sampling of "No Finding" Cases

"No Finding" represents approximately 40% of the dataset. To prevent bias toward negative predictions, we:

1. Separate "No Finding" samples from pathological samples.
2. Downsample "No Finding" cases to 35% of their original representation.
3. Combine the full set of pathological samples with the downsampled normal cases.
4. Shuffle the combined dataset to ensure random distribution across training batches.

## 3.2.6 Train-Validation Split

The preprocessed dataset is divided into training and validation sets:

1. 80% of the data is allocated to the training set.
2. 20% is reserved for validation.
3. Splitting is performed using stratified sampling to maintain relative class distributions across both sets.
4. A fixed random seed (42) ensures reproducibility of the split.

## 3.2.7 Preprocessing Pipeline Implementation

The entire preprocessing workflow is implemented in the `ChestXrayDataset` class, which extends PyTorch's `Dataset` class. This ensures:

1. On-the-fly preprocessing during training to minimize memory requirements.
2. Consistent preprocessing between training, validation, and inference.
3. Efficient handling of large datasets through lazy loading.
4. Graceful error handling for corrupted or missing images.

The preprocessing pipeline operates as a sequence of transformations:

```
Raw DICOM/JPG files → Resolution standardization → Intensity normalization →
Data augmentation → Metadata encoding → Label processing → Batch formation
```

In summary, our comprehensive preprocessing strategy addresses the unique challenges of chest X-ray analysis, enabling effective training of deep learning models for accurate multi-label classification and pathology localization.
