# توثيق شامل لنظام تحليل صور الأشعة السينية للصدر

## 1. نظرة عامة على المشروع

هذا المشروع عبارة عن نظام متكامل لتحليل صور الأشعة السينية للصدر باستخدام الذكاء الاصطناعي. يهدف النظام إلى مساعدة الأطباء والمهنيين الطبيين على تشخيص الأمراض المختلفة من خلال:
- تحميل صور الأشعة السينية
- تحليلها تلقائياً بواسطة نماذج التعلم العميق
- تحديد المناطق المشتبه بها (Bounding Boxes)
- إنشاء خرائط حرارية (Heatmaps) لتوضيح مناطق الاهتمام
- توفير تقارير مفصلة قابلة للطباعة (PDF)
- متابعة تاريخ المرضى والنتائج

## 2. هيكل المشروع

المشروع مكون من جزأين رئيسيين:

1. **واجهة أمامية (Frontend):** مبنية باستخدام Next.js و React
2. **خادم خلفي (Backend):** مبني باستخدام Python و FastAPI

## 3. المكونات التقنية

### 3.1 الواجهة الأمامية (Frontend)

#### 3.1.1 التقنيات المستخدمة
- **Next.js**: إطار عمل React للتطوير السريع
- **TypeScript**: لغة برمجة آمنة النوع
- **React**: مكتبة لبناء واجهات المستخدم
- **Tailwind CSS**: إطار عمل CSS للتصميم السريع
- **Shadcn UI**: مكتبات مكونات واجهة المستخدم
- **Framer Motion**: مكتبة للرسوم المتحركة

#### 3.1.2 الصفحات الرئيسية
1. **الصفحة الرئيسية**: نقطة الدخول للتطبيق
2. **لوحة التحكم**: عرض الإحصاءات والبيانات المهمة
3. **صفحة التحليل**: تحميل ومعالجة صور الأشعة
4. **صفحة النتائج**: عرض نتائج التحليل بتفصيل
5. **إدارة المرضى**: إضافة وتعديل بيانات المرضى
6. **التقارير**: توليد وعرض التقارير الطبية

#### 3.1.3 المكونات الرئيسية
1. **`heatmap-view.tsx`**:
   - عرض الخرائط الحرارية للأمراض المكتشفة
   - تفاعلي للتبديل بين الأمراض المختلفة
   - يمكن ضبط شفافية وطريقة عرض الخريطة الحرارية
   - يعرض مربعات التحديد (Bounding Boxes) للأمراض

2. **`model-results.tsx`**:
   - عرض نتائج تحليل النموذج بطريقة منظمة
   - يوفر علامات تبويب للتبديل بين النتائج المختلفة
   - يعرض احتمالية كل مرض والمناطق المتأثرة

3. **`pdf-report-generator.tsx`**:
   - إنشاء تقارير PDF مفصلة للمرضى
   - تضمين معلومات المريض والطبيب
   - عرض صور الأشعة مع الخرائط الحرارية لكل مرض
   - توفير معلومات التشخيص والتوصيات

4. **`interactive-uploader.tsx`**:
   - واجهة سهلة الاستخدام لتحميل صور الأشعة
   - دعم السحب والإفلات
   - معاينة الصور قبل الرفع
   - تتبع تقدم التحميل

### 3.2 الخادم الخلفي (Backend)

#### 3.2.1 التقنيات المستخدمة
- **Python**: لغة البرمجة الأساسية
- **FastAPI**: إطار عمل API سريع وحديث
- **PyTorch**: مكتبة للتعلم العميق
- **CUDA**: لتسريع حسابات التعلم العميق على GPU

#### 3.2.2 المكونات الرئيسية
1. **`model_server.py`**:
   - خادم FastAPI لمعالجة طلبات تحليل الصور
   - تحميل نماذج التعلم العميق وإدارتها
   - توفير نقطة نهاية API للتنبؤ
   - إنشاء خرائط حرارية للأمراض المختلفة

2. **`predict.py`**:
   - كود التنبؤ الأساسي
   - معالجة الصور وتنفيذ الاستدلال
   - تحديد المناطق المشتبهة (Bounding Boxes)

3. **نماذج التعلم العميق**:
   - نموذج تصنيف الأمراض المختلفة
   - نموذج تحديد المناطق المتأثرة
   - نموذج إنشاء الخرائط الحرارية

## 4. قاعدة البيانات

المشروع يستخدم **Supabase** كقاعدة بيانات وخدمة تخزين:

### 4.1 جداول قاعدة البيانات
1. **`patients`**: بيانات المرضى (الاسم، العمر، الجنس، إلخ)
2. **`analyses`**: تسجيلات تحليل الصور (تاريخ التحليل، رابط الصورة، إلخ)
3. **`results`**: نتائج التحليل (الأمراض المكتشفة، درجة الثقة، المناطق المحددة)
4. **`users`**: بيانات المستخدمين (الأطباء والمسؤولين)
5. **`profiles`**: معلومات إضافية للمستخدمين (التخصص، المستشفى، إلخ)

### 4.2 تخزين الملفات
- تخزين صور الأشعة الأصلية
- تخزين التقارير المنشأة (PDF)
- إدارة المرفقات والوثائق الطبية

## 5. تدفق العمل

### 5.1 تحميل وتحليل الصور
1. المستخدم يقوم بتسجيل الدخول إلى النظام
2. يدخل بيانات المريض أو يختار مريضاً موجوداً
3. يقوم بتحميل صورة الأشعة السينية
4. النظام يرسل الصورة إلى الخادم الخلفي للتحليل
5. يتم تحليل الصورة باستخدام نماذج التعلم العميق
6. النظام يعيد النتائج مع تحديد الأمراض المحتملة والمناطق المتأثرة

### 5.2 عرض النتائج
1. النظام يعرض نتائج التحليل في واجهة سهلة الفهم
2. يتم عرض الخرائط الحرارية لتسليط الضوء على المناطق المشبوهة
3. يمكن للطبيب اختيار مرض معين لعرض الخريطة الحرارية الخاصة به
4. يمكن ضبط شفافية وطريقة عرض الخرائط الحرارية
5. يتم عرض احتمالية كل مرض ومدى خطورته

### 5.3 إضافة ملاحظات وإنشاء تقارير
1. الطبيب يقوم بإضافة ملاحظات وتشخيص
2. يمكن مقارنة الصورة الحالية مع صور سابقة للمريض
3. إنشاء تقرير PDF يتضمن:
   - معلومات المريض والطبيب
   - الصورة الأصلية
   - خرائط حرارية لكل مرض مكتشف
   - نتائج التحليل وتفاصيل الاحتمالات
   - ملاحظات الطبيب والتوصيات

## 6. نموذج التعلم العميق

### 6.1 بنية النموذج
النظام يستخدم نموذجاً عميقاً معقداً مكون من:
1. **شبكة استخراج الميزات**: تستخرج الميزات المهمة من صور الأشعة
2. **آلية الانتباه (Attention Mechanism)**: تركز على المناطق المهمة في الصورة
3. **طبقات التصنيف**: تحدد الأمراض المختلفة واحتمالياتها
4. **طبقات التحديد**: تعين مربعات التحديد حول المناطق المتأثرة

### 6.2 الأمراض التي يمكن اكتشافها
النظام يستطيع اكتشاف عدة أمراض منها:
1. الالتهاب الرئوي (Pneumonia)
2. انصباب الجنب (Effusion)
3. تضخم القلب (Cardiomegaly)
4. انخماص الرئة (Atelectasis)
5. كتلة (Mass)
6. عقيدة (Nodule)
7. استرواح الصدر (Pneumothorax)
8. ارتشاح (Infiltration)
وغيرها...

## 8. المزايا الرئيسية

1. **دقة عالية**: نماذج تعلم عميق متقدمة لتحليل صور الأشعة
2. **التفسير البصري**: خرائط حرارية ومربعات تحديد لتوضيح الأمراض
3. **تخصيص العرض**: إمكانية التبديل بين الأمراض المختلفة وضبط طريقة العرض
4. **تقارير شاملة**: إنشاء تقارير PDF مفصلة مع الخرائط الحرارية لكل مرض
5. **تتبع المرضى**: حفظ تاريخ التحليلات والمقارنة بين الصور السابقة
6. **واجهة سهلة الاستخدام**: تصميم عصري وبديهي للمستخدمين الطبيين

## 9. كيفية تشغيل المشروع

### 9.1 متطلبات النظام
- **Node.js** (v14 أو أعلى)
- **Python** (3.8 أو أعلى)
- **CUDA** (اختياري لتسريع وحدة معالجة الرسومات)
- **حساب Supabase**

### 9.2 تثبيت وتشغيل الواجهة الأمامية
```bash

# تشغيل خادم التطوير
npm run dev
```

### 9.3 تثبيت وتشغيل الخادم الخلفي
```bash

.venv\Scripts\activate     # لنظام Windows

pip install -r requirements.txt
```
